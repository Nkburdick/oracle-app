# Phase 3 — PWA Install + Service Worker + Push Notifications

## Overview

Make Oracle App installable as a Progressive Web App with push notification support. This is the #3 item on the Telegram exit critical path — iOS only supports Web Push for home-screen-installed PWAs (iOS 16.4+), so this PR directly unblocks the notifications feed and the Telegram sunset.

**Pennyworth backend is 100% ready.** VAPID configured, subscribe/unsubscribe/send-test endpoints all shipped and tested (pennyworth#38). This PR is Oracle App frontend only.

**One PR** — manifest, service worker, push subscription, icons, install prompt, meta tags, and app shell caching. All interconnected.

## Architecture

### Current State

```
Oracle App (SvelteKit + adapter-node)
├── No manifest
├── No service worker
├── No icons (only Svelte logo favicon)
├── No push subscription code
└── No PWA meta tags
```

### Target State

```
Oracle App (SvelteKit + adapter-node + @vite-pwa/sveltekit)
├── manifest.webmanifest (generated by VitePWA)
├── Service worker (injectManifest — custom push + Workbox precaching)
├── Darkmatter icon set (192, 512, apple-touch-icon, favicon)
├── Push subscription client (VAPID subscribe/re-subscribe)
├── iOS install prompt (bottom sheet with Add to Home Screen instructions)
├── PWA meta tags (theme-color, apple-mobile-web-app-*)
└── App shell caching (HTML, CSS, JS, fonts — network-first for API)
```

### Push Flow

```
[Oracle App installed on iOS home screen]
  ↓ User taps "Enable notifications" after first chat response
  ↓ navigator.serviceWorker.ready → pushManager.subscribe(VAPID key)
  ↓ POST /api/push/subscribe → Pennyworth stores subscription
  ...later...
  ↓ Pennyworth sends push via web-push library
  ↓ Service worker receives push event
  ↓ self.registration.showNotification() (REQUIRED on iOS — no silent push)
  ↓ User taps notification → notificationclick → open/focus Oracle App
```

### 2. Service Worker (Custom — `src/sw.ts`)

```typescript
/// <reference lib="webworker" />
import { precacheAndRoute } from 'workbox-precaching';

declare const self: ServiceWorkerGlobalScope;

// Workbox injects precache manifest here at build time
precacheAndRoute(self.__WB_MANIFEST);

// ─── Push Handler ──────────────────────────────────────────────────────────
// iOS REQUIRES showing a notification for every push event.
// Silent pushes will cause iOS to revoke push permission.

self.addEventListener('push', (event) => {
	const data = event.data?.json() ?? {};
	const options: NotificationOptions = {
		body: data.body ?? '',
		icon: '/icon-192.png',
		badge: '/icon-192.png',
		tag: data.tag ?? 'oracle-default',
		data: { url: data.url ?? '/' }
	};

	event.waitUntil(self.registration.showNotification(data.title ?? 'Oracle', options));
});

// ─── Notification Click ────────────────────────────────────────────────────
// Open or focus the app when the user taps a notification.

self.addEventListener('notificationclick', (event) => {
	event.notification.close();
	const url = event.notification.data?.url ?? '/';

	event.waitUntil(
		self.clients.matchAll({ type: 'window', includeUncontrolled: true }).then((clients) => {
			const existing = clients.find((c) => new URL(c.url).pathname === url);
			if (existing) return existing.focus();
			return self.clients.openWindow(url);
		})
	);
});
```

### 5. Push Subscription Client

**File:** `src/lib/push.ts`

Client-side module for managing push subscriptions:

```typescript
/** Check if push is supported and the app is installed as PWA */
export function isPushSupported(): boolean {
	return (
		'serviceWorker' in navigator &&
		'PushManager' in window &&
		window.matchMedia('(display-mode: standalone)').matches
	);
}

/** Get the VAPID public key from Pennyworth (via Oracle App proxy) */
async function getVapidKey(): Promise<string> {
	const res = await fetch('/api/push/vapid-public-key');
	if (!res.ok) throw new Error('Failed to get VAPID key');
	const { publicKey } = await res.json();
	return publicKey;
}

/** Subscribe to push notifications */
export async function subscribePush(): Promise<PushSubscription> {
	const registration = await navigator.serviceWorker.ready;
	const vapidKey = await getVapidKey();

	const subscription = await registration.pushManager.subscribe({
		userVisibleOnly: true,
		applicationServerKey: vapidKey
	});

	// Send subscription to Pennyworth
	const res = await fetch('/api/push/subscribe', {
		method: 'POST',
		headers: { 'Content-Type': 'application/json' },
		body: JSON.stringify({
			endpoint: subscription.endpoint,
			keys: {
				p256dh: btoa(String.fromCharCode(...new Uint8Array(subscription.getKey('p256dh')!))),
				auth: btoa(String.fromCharCode(...new Uint8Array(subscription.getKey('auth')!)))
			},
			userAgent: navigator.userAgent
		})
	});

	if (!res.ok) throw new Error('Failed to register subscription');
	return subscription;
}

/** Re-subscribe if existing subscription is stale (call on every app launch) */
export async function ensureSubscription(): Promise<void> {
	if (!isPushSupported()) return;
	const registration = await navigator.serviceWorker.ready;
	const existing = await registration.pushManager.getSubscription();
	if (existing) {
		// Confirm subscription is still valid with server
		await fetch('/api/push/subscribe', {
			method: 'POST',
			headers: { 'Content-Type': 'application/json' },
			body: JSON.stringify({
				endpoint: existing.endpoint,
				keys: {
					p256dh: btoa(String.fromCharCode(...new Uint8Array(existing.getKey('p256dh')!))),
					auth: btoa(String.fromCharCode(...new Uint8Array(existing.getKey('auth')!)))
				}
			})
		});
	}
}
```

### 6. Push API Proxy Routes

**New routes in Oracle App** (proxying to Pennyworth, same pattern as chat proxy):

- `GET /api/push/vapid-public-key` → Pennyworth `GET /api/push/vapid-public-key`
- `POST /api/push/subscribe` → Pennyworth `POST /api/push/subscribe`
- `POST /api/push/unsubscribe` → Pennyworth `POST /api/push/unsubscribe`

These are simple proxies — read request, forward with Bearer auth, return response.

- "Get push notifications and a native app experience"
- Dismiss button

### 8. Push Permission Prompt

**Trigger:** After the first chat response is received (in both ProjectChats.svelte and mobile +page.svelte).

**Flow:**

1. After first assistant message arrives, check `isPushSupported()`
2. If supported and not yet subscribed, show a contextual prompt:
   "Enable notifications so you never miss Alfred's replies"
3. On "Enable" → call `subscribePush()`
4. On "Not now" → dismiss, don't ask again for 30 days
5. Store state in localStorage
