  • RSVP system working for events/meets
  • Parent read-only athlete view functional
  • Team join link and invite flow working
  • Mike Finley actively using StrideMind weekly with his team

  • Polish based on Mike's feedback from Phase 1
  • Onboarding flow for new coaches (self-serve)
  • 3-5 additional coaches actively using StrideMind
  • Pricing and payment flow live
  • Marketing site / landing page

Decision Gate: Native Apps

  • Evaluate PWA limitations based on real usage data
  • Decision: stay PWA or build native iOS (then Android)
  • Only proceed if PWA hits a wall on push, offline, or trust

Project Complete When:

  • 5+ coaches actively using StrideMind with their teams
  • Revenue being generated (any amount)
  • At this point, promote to Area with ongoing sub-projects

Scope

In Scope (Phase 1 — NOW)

  • Team communication: announcements, push notifications, delivery confirmation
  • Scheduling: events, schedule changes, RSVPs
  • Parent views: read-only athlete view, join link, family invite
  • PWA push notification reliability (the #1 technical priority)

  • Pricing/payment integration

  • Social media presence management (Phase 2+)
  • Multi-sport support (running only for now)

Context

StrideMind is a youth running coach app that combines training management with team communication. Built as a Next.js PWA. MVP deployed and being beta tested by Coach Mike Finley (Jonah Burdick's cross-country coach), but Mike isn't using it heavily yet.

Nick realized StrideMind needs TeamSnap-like team communication features to be genuinely useful — coaches don't just need training tools, they need a reliable way to reach parents. The Council debate (2026-03-16) produced unanimous agreement: comms-first, PWA-only, ship fast, measure delivery confirmation as the single metric that matters.

Key Insight from Council

"If a parent says 'I never got that alert,' the app has already failed and you will never get that parent's trust back." — Delivery confirmation is the trust foundation. Everything else is Phase 2.

First Actions

  1. Schedule observation session — watch Mike use TeamSnap, note friction points
  2. Triage 27 open GitHub issues — identify the 6-8 that satisfy Phase 1 scope
  3. Ship Team Comms MVP in 2-3 weeks
  4. Measure: does Mike open StrideMind weekly?

Key Decisions

  • 2026-03-16: Comms-first MVP — training tools wait for Phase 2 (Council unanimous)
  • 2026-03-16: PWA-only — native apps are a decision gate, not a phase
  • 2026-03-16: Delivery confirmation is THE metric for Phase 1
  • 2026-03-16: Keep as Project, promote to Area when 5+ coaches and revenue
  • 2026-03-16: Observe Mike using TeamSnap before writing more code

Current Status

Active development on Team Communication features. Phase 1 comms MVP tasks in Todoist with dates through May 2026. Onboarding wizard, beta invite simplification, and PWA offline caching are the priority build items. White-glove session with Mike Finley is the key unlock.
